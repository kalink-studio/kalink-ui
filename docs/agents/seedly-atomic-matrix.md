# Seedly Atomic Matrix

This matrix is the canonical atomic-design mapping for the public Seedly surfaces in `packages/seedly-react` and `packages/seedly`.

## Layer model

- `reset -> base -> atoms -> molecules -> organisms -> layouts -> utilities -> overrides`
- `reset` is reserved for normalization rules; `base` is reserved for system defaults such as typography and theme globals.
- Responsive helpers should stay in the same layer as their non-responsive counterpart.
- `Heading` is intentionally classified as a molecule because it composes `Text`.
- `ContextMenu` and `Menubar` should expose their dependency on `Menu` clearly; compose shared popup pieces from `Menu.*`.

## Internal foundations

| Module             | Path                                                        | Notes                                                                                                                                                          |
| ------------------ | ----------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Layout foundation  | `packages/seedly/src/components/layout/`                    | Internal structural foundation for layout primitives.                                                                                                          |
| Shared foundations | `packages/seedly/src/components/_foundation/`               | Keep helper seams explicit; helpers should inherit the owning component layer instead of hard-coding their own.                                                |
| React internals    | `packages/seedly-react/src/components/_internal/`           | Internal render helpers, not public components.                                                                                                                |
| Range track helper | `packages/seedly/src/components/_foundation/range-track.ts` | Pure TypeScript helper module; consuming components own their contracts and pass concrete vars into `createRangeTrackStyles` and `createRangeIndicatorStyles`. |

## Global style ownership

- Layer every `globalStyle(...)` call by the owning component or foundation, not by selector shape alone.
- `packages/seedly/src/styles/reset.css.ts` belongs in `reset`.
- `packages/seedly/src/styles/typography.css.ts` and `packages/seedly/src/styles/typography.responsive.css.ts` belong in `base`.
- Atom-owned descendants such as `packages/seedly/src/components/button/icon.css.ts` stay in `atoms`.
- Molecule-owned descendants such as `packages/seedly/src/components/checkbox/checkbox.css.ts` and `packages/seedly/src/components/switch/switch.css.ts` stay in `molecules`.
- Layout-owned child selectors such as `packages/seedly/src/components/cover/cover.css.ts`, `packages/seedly/src/components/sidebar/sidebar.css.ts`, and `packages/seedly/src/components/switcher/switcher.css.ts` stay in `layouts`.
